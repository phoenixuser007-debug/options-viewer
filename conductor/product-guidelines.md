# Product Guidelines

## Visual Identity
**Style:** Data-Dense Modern
- **Theme:** High-contrast dark mode is the default and primary interface to reduce eye strain during extended use.
- **Typography:** Use monospaced fonts for all numerical data to ensure perfect vertical alignment and readability. Sans-serif fonts for labels and navigation.
- **Density:** Maximize screen real estate usage. Compact rows and reduced padding to display as many strike prices as possible without scrolling.

## Design Principles
**Precision:**
- **Alignment:** All numerical columns must be right-aligned. Text columns left-aligned. Decimal points should vertically align.
- **Formatting:** Consistent usage of decimal places (e.g., 2 for prices, 4 for Greeks).
- **Color Coding:** Use standard financial colors (Green for up/positive, Red for down/negative) but ensure accessible contrast ratios against the dark background.
- **Scannability:** Alternating row backgrounds (zebra striping) to help users track data across wide tables.

## User Experience
- **Responsiveness:** The UI should remain fluid even with high-frequency data updates.
- **Clarity:** Tooltips and clear headers should explain complex financial abbreviations (IV, OI, etc.).
